---
description: 프로젝트 규칙에 따른 커밋 메시지 생성 (한글)
---

1. 변경된 파일 목록을 확인합니다 (`git status`).
2. 변경 내용을 바탕으로 프로젝트의 커밋 메시지 규칙(`[type]: [description]`)을 준수하는 **한글** 커밋 메시지를 생성합니다.
3. 생성된 커밋 메시지를 출력합니다.

**주의사항:**

- **반드시 한글로 작성해야 합니다.**
- 커밋 메시지 형식: `[type]: [description]`
- Type 종류: `feat`, `fix`, `style`, `refactor`, `chore`, `docs`, `design`
- 예시: `[design]: 메인 페이지 배너 스타일 수정`
